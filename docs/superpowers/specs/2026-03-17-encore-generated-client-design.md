# Encore Generated Client Migration

**Date:** 2026-03-17
**Status:** Approved

## Goal

Replace the hand-written `api.ts` fetch wrapper and manually-mirrored TypeScript types with Encore's generated type-safe client. This eliminates type drift between frontend and backend.

## Context

The current frontend uses a hand-written `src/lib/api.ts` that:
- Wraps `fetch` with a generic `request<T>()` function
- Manually mirrors all backend response types as TypeScript interfaces
- Routes all calls through the Vite dev proxy (`/api/* → localhost:4000`)

Encore supports generating a TypeScript client via `encore gen client`, which produces typed service clients that stay in sync with the backend automatically.

## Deployment Context

- Coolify + Docker, frontend and backend as separate containers
- Same domain: `example.com/` (frontend), `example.com/api/` (backend via nginx)
- Dev: Vite proxy `/api/* → localhost:4000` (strips `/api` prefix before forwarding)
- No CORS configuration needed (same-domain routing in both environments)
- Known constraint: the absolute SSE URL (`window.location.origin + '/api/...'`) assumes the app is served at the root path. Sub-path deployments (e.g., `example.com/app/`) would require a different approach.

## File Structure

```
frontend/apps/web/src/lib/
├── client.gen.ts     ← generated by encore gen client, committed to git
├── client.ts         ← hand-written singleton initializer (replaces api.ts)
├── auth.ts           ← unchanged
└── api.ts            ← deleted after migration is complete and TypeScript compiles cleanly
```

## BaseURL Strategy

Use `window.location.origin + '/api'` — no environment variable needed:

```typescript
// src/lib/client.ts
import { getToken } from "./auth";
import Client from "./client.gen";

export const client = new Client(
    `${window.location.origin}/api`,
    // auth is called per-request (lazy), not captured at construction time
    { auth: () => getToken() ?? "" }
);
```

- Dev: `http://localhost:3001/api` → Vite proxy strips `/api` → `localhost:4000`
- Prod: `https://example.com/api` → nginx strips `/api` → backend container

The Vite proxy config and nginx config remain unchanged.

**Verification required:** After generating `client.gen.ts`, confirm the generated `BaseClient` concatenates `baseURL + path` where path starts with `/` (e.g., `baseURL=/api` + `/subscriptions` = `/api/subscriptions`). Do this before deleting `api.ts`.

**Auth on public endpoints:** The generated client sends `Authorization: Bearer ` with an empty string for unauthenticated requests (when `getToken()` returns null). Encore's auth handler only runs for `//encore:api auth` endpoints — public endpoints ignore the Authorization header entirely. This is an intentional, accepted difference from the current behavior (which omits the header).

## Error Handling

The generated client throws errors with `code`, `message`, and `status` fields. Replace `ApiError` class usage with a tighter type guard:

```typescript
// src/lib/client.ts
export function isApiError(err: unknown): err is { code: string; message: string; status: number } {
    return (
        typeof err === "object" &&
        err !== null &&
        "code" in err &&
        "message" in err &&
        "status" in err &&
        typeof (err as Record<string, unknown>).status === "number"
    );
}
```

Route files: replace `err instanceof ApiError` checks with `isApiError(err)`.

## SSE (Progress Stream)

`EventSource` does not support custom headers. The SSE endpoint (`/check/:jobId/progress`) is declared `//encore:api public raw` and requires no auth — maintain this behavior. Update URL construction to use absolute URL for consistency:

```typescript
// old
new EventSource(`/api/check/${jobId}/progress`)

// new
new EventSource(`${window.location.origin}/api/check/${jobId}/progress`)
```

The generated client's `GetProgress` method (returns raw `Response`) is not used — `EventSource` is kept.

## Type Field Naming

The generated client preserves JSON tag names from Go structs:
- **Response/entity types** use snake_case (matching JSON tags): `result.node_name`, `job.subscription_id`, `result.latency_ms`
- **Query parameter structs** use PascalCase Go field names: `{ JobID: jobId }`, `{ Limit: 10, Offset: 0 }`

This means property reads on response objects remain unchanged from hand-written types. Only query parameter construction uses PascalCase.

## Type Imports

Delete all hand-written interfaces from `api.ts`. Import types from the generated client namespaces:

```typescript
import type { checker } from "@/lib/client.gen"
type NodeResult = checker.NodeResult
```

## Call Site Migration

All 28 call sites across 7 files are migrated from `api.get/post/put/delete()` to typed service client methods. Migrate file by file, keep `api.ts` until all files compile cleanly, then delete it.

```typescript
// old
api.get<ResultsResponse>(`/check/${id}/results?job_id=${jobId}`)

// new
client.checker.GetResults(id, { JobID: jobId })   // query param = PascalCase
// response fields unchanged: result.node_name, result.latency_ms, etc.
```

## Regeneration

Add to `frontend/apps/web/package.json` (run from `frontend/apps/web/` to avoid path ambiguity):

```json
"gen:client": "encore gen client subs-check-uqti --lang=typescript --output=./src/lib/client.gen.ts"
```

Or add to `frontend/package.json` with an explicit path:

```json
"gen:client": "encore gen client subs-check-uqti --lang=typescript --output=./apps/web/src/lib/client.gen.ts"
```

Run when backend API changes. The generated file is committed to git so Docker builds do not require the Encore CLI.
