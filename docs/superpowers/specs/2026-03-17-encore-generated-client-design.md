# Encore Generated Client Migration

**Date:** 2026-03-17
**Status:** Approved

## Goal

Replace the hand-written `api.ts` fetch wrapper and manually-mirrored TypeScript types with Encore's generated type-safe client. This eliminates type drift between frontend and backend.

## Context

The current frontend uses a hand-written `src/lib/api.ts` that:
- Wraps `fetch` with a generic `request<T>()` function
- Manually mirrors all backend response types as TypeScript interfaces
- Routes all calls through the Vite dev proxy (`/api/* → localhost:4000`)

Encore supports generating a TypeScript client via `encore gen client`, which produces typed service clients that stay in sync with the backend automatically.

## Deployment Context

- Coolify + Docker, frontend and backend as separate containers
- Same domain: `example.com/` (frontend), `example.com/api/` (backend via nginx)
- Dev: Vite proxy `/api/* → localhost:4000`
- No CORS configuration needed (same-domain routing in both environments)

## File Structure

```
frontend/apps/web/src/lib/
├── client.gen.ts     ← generated by encore gen client, committed to git
├── client.ts         ← hand-written singleton initializer (replaces api.ts)
├── auth.ts           ← unchanged
└── api.ts            ← deleted after migration
```

## BaseURL Strategy

Use `window.location.origin + '/api'` — no environment variable needed:

```typescript
// src/lib/client.ts
import { getToken } from "./auth";
import Client from "./client.gen";

export const client = new Client(
    `${window.location.origin}/api`,
    { auth: () => getToken() ?? "" }
);
```

- Dev: `http://localhost:3001/api` → Vite proxy → `:4000`
- Prod: `https://example.com/api` → nginx → backend container

The Vite proxy config and nginx config remain unchanged.

## Error Handling

The generated client throws errors in its own format. Replace `ApiError` class usage with a type guard:

```typescript
// src/lib/client.ts
export function isApiError(err: unknown): err is { code: string; message: string; status: number } {
    return typeof err === "object" && err !== null && "code" in err;
}
```

Route files: replace `err instanceof ApiError` checks with `isApiError(err)`.

## SSE (Progress Stream)

`EventSource` does not support custom headers. The current SSE endpoint (`/check/:jobId/progress`) already works without auth headers — maintain this behavior. Update URL construction to match the client's baseURL source:

```typescript
// old
new EventSource(`/api/check/${jobId}/progress`)

// new
new EventSource(`${window.location.origin}/api/check/${jobId}/progress`)
```

The generated client's `GetProgress` method (returns raw `Response`) is not used — `EventSource` is kept.

## Type Imports

Delete all hand-written interfaces from `api.ts`. Import types from the generated client namespaces:

```typescript
import type { checker } from "@/lib/client.gen"
type NodeResult = checker.NodeResult
```

## Call Site Migration

All 28 call sites across 7 files are migrated from `api.get/post/put/delete()` to typed service client methods:

```typescript
// old
api.get<ResultsResponse>(`/check/${id}/results?job_id=${jobId}`)

// new
client.checker.GetResults(id, { JobID: jobId })
```

## Regeneration

Add to `frontend/package.json`:

```json
"gen:client": "encore gen client subs-check-uqti --lang=typescript --output=./apps/web/src/lib/client.gen.ts"
```

Run from `frontend/` when backend API changes. The generated file is committed to git so Docker builds do not require the Encore CLI.
